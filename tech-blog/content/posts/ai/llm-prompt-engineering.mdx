# LLM 프롬프트 엔지니어링 실전 가이드

효과적인 프롬프트 작성은 AI 활용의 핵심입니다.

## 프롬프트 엔지니어링 기본 원칙

### 1. 명확한 지시

모호한 표현을 피하고 구체적으로 요청합니다.

```
❌ "코드 작성해줘"
✅ "Python으로 CSV 파일을 읽어 JSON으로 변환하는 함수를 작성해줘"
```

### 2. 컨텍스트 제공

배경 정보를 충분히 제공하여 정확한 응답을 유도합니다.

### 3. 출력 형식 지정

원하는 결과물의 형식을 명시합니다.

```
다음 형식으로 응답해주세요:
- 요약: (한 문장)
- 핵심 포인트: (3개의 불릿 포인트)
- 결론: (한 문장)
```

## 고급 기법

### Chain of Thought (CoT)

단계별 추론을 유도하여 복잡한 문제를 해결합니다.

```
문제를 단계별로 분석해주세요:
1. 먼저 문제를 이해합니다
2. 필요한 정보를 파악합니다
3. 해결 방안을 도출합니다
4. 최종 답변을 제시합니다
```

### Few-shot Learning

예시를 제공하여 원하는 패턴을 학습시킵니다.

## 결론

프롬프트 엔지니어링은 반복적인 실험과 개선을 통해 발전합니다. 다양한 기법을 시도하고 결과를 분석하여 최적의 프롬프트를 찾아가세요.
