# 자연스러운 기술 블로그 작성 프롬프트 가이드 (개선판)

## 핵심 원칙

### 1. 톤 & 문체 설정
```
- 문장 길이를 다양하게 섞기 (짧은 문장 + 긴 문장 교차)
- 구어체와 문어체의 중간 지점 유지
- 일상어 위주로 작성하되, 필요한 기술 용어는 간단히 설명
```

### 2. AI 스러운 표현 제거 규칙

**피해야 할 표현** 
- "가장 ~한", "바로", "정말", "특히" 등 과도한 강조 부사
- "~하는데요", "~죠" 등 과도하게 친근한 어미
- "기억하세요", "활용해보세요" 등 명령형 문장
- "쉽게 말해", "자유자재로", "완벽하게" 등 과장된 수식어
- "과연 ~일까요?", "~하는데, 정말" 등 의문형 남발

**권장 표현** 
- "~합니다", "~입니다" 형태의 담담한 평서문
- "즉", "다시 말해", "예를 들어" 등 논리적 연결어
- "차이가 있습니다", "방법이 다릅니다" 등 사실 기반 서술

### 3. 글 구조 (실전 베스트 프랙티스)
**도입부**
```
- 1-2 문장으로 주제를 간결하게 소개
- 개인적 경험이나 구체적 상황으로 시작 (선택사항)
- 과장된 문제 제기 피하기
- 이 글에서 다룰 내용을 명시

예시:
"Docker Compose에서 자주 사용하는 네트워크, 볼륨, 포트 매핑을 정리합니다."
```

**본문**
```
- 코드 블록은 반드시 언어 명시 (syntax highlighting)
- 코드 설명 시 특정 라인이나 변수를 구체적으로 언급
- 스크린샷이나 다이어그램 활용 (필요시)
- 단계별 설명은 논리적 순서로 배치
- 불필요한 마크다운/불릿 포인트 남발 지양 - 자연스러운 문단 흐름 유지
```

**마무리**
```
- 명령형 지시 문장 제거 ("기억하세요" ❌)
- 핵심 개념 1-2줄로 간단히 요약
- 추가 리소스나 참고 링크 제공 (선택사항)

예시:
"서비스 이름을 통한 네트워크 통신과 Named Volume 정의 방식이 핵심입니다."
```

### 4. 강조 요소 사용법

```
💡 실무 팁 → 💡 팁
💡 기억하세요 → 💡 중요 또는 💡 참고
💡 주의사항 → 💡 주의

- 이모지는 최소한으로 사용
- 강조 박스는 정말 중요한 내용만
```

### 5. 문장 스타일 가이드

**문장 길이 및 리듬**
```
- 짧은 문장(10-15단어)과 긴 문장(20-30단어)을 교차 배치
- 모든 문장이 같은 구조면 단조로움 → 다양한 구조 사용
- 한 문단은 3-5 문장 이내로 유지
```

**단어 선택**
```
- 같은 단어 반복 피하기 (동의어 활용)
- 기술 용어는 첫 등장 시 한 번만 설명
- 축약형 자연스럽게 사용 가능 (할 수 있습니다 → 할 수 있다)
```

**자연스러운 흐름** 
```
- 문단 간 자연스러운 전환 문구 사용
- 불릿 포인트로 나열하기보다 문장으로 연결
- 독자가 막히지 않고 읽을 수 있는 흐름 유지
```

### 6. 초안 수정 체크리스트

AI가 생성한 초안을 다음 기준으로 검토:

```
□ 똑같은 문장 구조가 3번 이상 반복되는가?
□ "바로", "정말", "특히" 같은 부사가 과도한가?
□ 독자에게 명령하는 톤이 있는가?
□ 문단이 너무 짧거나(1-2줄) 너무 긴가(7줄 이상)?
□ 코드 예제에 충분한 설명이 있는가?
□ 도입부가 과장되거나 불필요하게 긴가?
□ 마무리에 "기억하세요", "활용해보세요" 같은 표현이 있는가?
```

## 실전 프롬프트 템플릿

### 기본 템플릿
```
당신은 5년차 개발자가 작성하는 기술 블로그 글을 작성하는 역할입니다.

[주제]: {작성할 주제}
[독자]: {초급/중급/고급 개발자}
[목적]: {튜토리얼/개념 설명/문제 해결 사례}

다음 규칙을 엄격히 준수해주세요:

**문체**
- 친구에게 설명하듯 작성하되 전문성 유지
- 짧은 문장(10-15단어)과 긴 문장(20-30단어) 교차 사용
- "~합니다", "~입니다" 형태의 평서문 중심

**절대 사용 금지 표현**
- 과도한 강조: "가장", "바로", "정말", "특히", "완벽하게"
- 명령형: "기억하세요", "활용해보세요", "알아보겠습니다"
- 과장: "자유자재로", "쉽게", "간단하게"
- 친근함 과다: "~죠", "~하는데요"

**구조**
- 도입부: 1-2문장으로 주제 소개 (과장 금지)
- 본문: 코드 예제 + 구체적 설명, 자연스러운 문단 흐름
- 마무리: 핵심 개념만 1-2줄로 요약 (명령문 금지)

**강조 요소**
- 💡 팁: (실무 팁 ❌)
- 💡 중요: (기억하세요 ❌)
- 💡 주의: (주의사항 ❌)

이제 위 규칙에 따라 글을 작성해주세요.
```

### 스타일 미러링 템플릿 [justdone](https://justdone.com/blog/writing/prompts-to-humanize-ai-text)
```
다음은 내가 이전에 작성한 블로그 글입니다:

[2-3개 문단 샘플 붙여넣기]

위 글의 톤, 리듬, 문장 구조를 학습하여 동일한 스타일로 다음 주제에 대해 작성해주세요:
[새로운 주제]

핵심 요구사항:
- 위 샘플과 동일한 문장 길이 패턴 사용
- 동일한 수준의 친근함과 전문성 유지
- 비슷한 단어 선택과 표현 방식 적용
```

### 수정 요청 템플릿 [blog.naver](https://blog.naver.com/zero_one_01/223812850743)
```
다음 초안을 더 자연스럽게 수정해주세요:

[AI가 생성한 초안]

수정 방향:
1. 반복되는 문장 구조 다양화
2. "바로", "정말", "특히" 같은 부사 제거
3. 명령형 문장을 평서문으로 변경
4. 과도하게 긴 문단(5문장 이상)은 분리
5. 마크다운 불릿 포인트 남발하지 말고 문단으로 풀어쓰기

특히 도입부와 마무리 부분에서 과장된 표현을 제거해주세요.
```
